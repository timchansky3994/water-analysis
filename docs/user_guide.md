# Руководство пользователя

Система анализа качества питьевой воды — инструмент поддержки решения специалиста‑гигиениста. Она помогает разобраться в выгрузках результатов лабораторных исследований и испытаний качества питьевой воды, выполненных в рамках социально-гигиенического мониторинга, и производственного контроля, проводимого ресурсоснабжающими организациям: проверяет полноту данных, ищет зависимости между показателями, сравнивает модели с простыми ориентирами и формирует отчёт с честным описанием ограничений.

Программа **не заменяет результаты лабораторных исследований и испытаний**. Расчётные оценки, которые она выдаёт, — это аналитические прогнозы модели, а не официальные результаты анализа, и они не должны записываться поверх исходных данных.

## Установка и запуск

Этот раздел нужен один раз — при первой установке. Если программа уже запущена, переходите к работе.

**Шаг 1. Проверьте Python.** Нужна версия 3.10 или новее:

```
python --version
```

**Шаг 2. Установите программу:**

```powershell
cd C:\путь\к\water-analysis
python -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install --upgrade pip
python -m pip install -e .[dev,ui]
```

**Шаг 3. Запустите интерфейс:**

```powershell
water-analysis ui
```

Браузер откроется сам. Если нет — откройте `http://localhost:8501`. Закрыть программу можно кнопкой «Завершить приложение» в боковом меню или сочетанием `Ctrl+C` в окне, из которого она запущена.

## Подготовка файла данных

Программа принимает лабораторные выгрузки CSV или XLSX. **Обязательные столбцы** (без них файл не будет обработан):

| Столбец | Пример значения |
|---------|-----------------|
| Дата проведения исследования | 15.03.2023 |
| Код точки | 10000000001.10110.0010 |
| Гигиенический показатель | Жесткость общая |
| Результат исследования | 4,2 |

Разные поставщики данных могут выгружать данные с разными именами столбцов, содержащих данные одного и того же вида (например, "Дата проведения исследования" vs "Дата"). В качестве основного (по умолчанию) взят формат, рекомендуемый отделениями Роспотребнадзора. Для удобства комплекс поддерживает создание своих **профилей источника** (`configs/source_profiles/*.yaml`). Можно переименовывать столбцы выгрузок по стандарту, представленному в таблице выше, а можно создать свой YAML по образцу `_default.yaml`, где явно указать свои названия столбцов. Кроме формата по умолчанию в исходном коде представлен ещё один (`configs/source_profiles/secondary.yaml`) для демонстрационных целей.

При загрузке файла программа показывает определённый профиль, период наблюдений, число строк и ОКТМО, а также превью первых 20 строк — проверьте, что данные прочитаны верно.

## Основные понятия

| Термин                        | Что означает                                                                                               |
|-------------------------------|------------------------------------------------------------------------------------------------------------|
| **ОКТМО**                     | код муниципального образования; первая часть кода точки                                                    |
| **Код точки**                 | полный идентификатор `ОКТМО.ТипТочки.НомерТочки`; сохраняется целиком                                      |
| **Тип точки**                 | тип точки отбора проб воды: 10110 — распределительная сеть, 10150 — водопровод, 10310/10320 — водоисточник |
| **Целевой показатель**        | показатель, который анализируется или оценивается (например, «Жесткость общая»)                            |
| **Предиктор**                 | сопутствующий показатель, по которому модель предсказывает целевой                                         |
| **Сценарий анализа**          | как объединяются точки в один срез (вся выборка, по ОКТМО, отдельная точка и т. д.)                        |
| **Цензурированное значение**  | результат вида `<0.05` или `>1.0` — известна только граница, не точное число                               |
| **Базовая модель (baseline)** | простой ориентир, например медиана или последнее значение; с ним сравнивают ML                             |
| **Holdout**                   | проверка на конечном отрезке данных, не участвовавшем в обучении                                           |
| **(Rolling) backtest**        | ретроспективная проверка на нарастающем отрезке данных с проверкой на следующем отрезке                    |
| **Расчётная оценка**          | прогноз модели для отсутствующего значения; не лабораторное измерение                                      |

## Анализ и отчёт

Основной сценарий. Раздел **«Анализ и отчёт»** повторяет шаги команды `report`.

**1. Загрузите файл.** Нажмите «Загрузите файл данных» и выберите CSV или XLSX. Программа определит профиль и покажет превью.

**2. Задайте параметры анализа:**

Сценарий анализа - Например, «По ОКТМО + Питьевая вода (10110 + 10150)» для питьевой воды
ОКТМО - выберите нужный из списка 
Целевой показатель - выберите из списка или впишите свой

В блоке «Дополнительные настройки» можно изменить список ML‑моделей, пороги пригодности и вес holdout/backtest в итоговой оценке. Для большинства задач их можно не трогать.

**3. Выберите предикторы.** Доступны три режима:

- **Автоматический** — программа сама отбирает показатели по корреляции и покрытию; настраиваются пороги.
- **Полу‑автоматический** — вы указываете обязательные показатели, остальные программа добавит сама.
- **Ручной** — в модель войдут только указанные вами показатели, корреляция не проверяется.

**4. Запустите анализ.** Нажмите «Запустить анализ». Под кнопкой отображается прогресс: профилирование, проверка пригодности, корреляции, сравнение моделей, сохранение таблиц, построение графиков. Время — от секунд до нескольких минут.

**5. Изучите результаты.** Сверху — цветной бейдж пригодности данных (зелёный/жёлтый/красный) с раскрывающимся списком причин и ограничений. Ниже — вкладки:

- **Сводка** — краткий отчёт для специалиста: пригодность, найденные связи, качество модели против базовой, предупреждения.
- **Таблицы** — детальные данные (пропуски, корреляции, сравнение моделей); каждую можно скачать в Excel.
- **Графики** — предсказанные значения, остатки, бэктест, тепловые карты.
- **Лог запуска** — технический журнал для диагностики.

Если модель обучена и пригодна, появится кнопка **«Перейти к расчёту прогнозируемых значений»**.

## Паспорт данных

Раздел **«Паспорт данных»** даёт быструю диагностику структуры файла без обучения модели.

1. Загрузите файл.
2. Выберите сценарий анализа (целевой показатель здесь не нужен).
3. Нажмите «Построить паспорт данных».

Результат по каждому срезу: число записей, ОКТМО, точек и показателей, а также вкладки — наблюдения по показателям, пропуски, покрытие по типам точек, постоянные ряды и матрица совместной встречаемости (как часто показатели измерены в один день). Таблицы можно скачать.

**Сезонный анализ.** Если в дополнительных настройках указать целевой показатель и гранулярность (по сезонам или месяцам), программа проверит наличие сезонного паттерна (критерий Краскела–Уоллиса) и построит сезонный профиль.

---

## Корреляции

Раздел **«Корреляции»** ищет статистические связи без обучения модели.

1. Загрузите файл и выберите сценарий.
2. Укажите целевые показатели, методы (Спирмен/Пирсон), минимум совместных измерений и число выводимых связей.
3. Нажмите «Найти зависимости».

Результат — таблица сильнейших корреляций и тепловая карта. Найденные показатели можно отметить и нажать **«Перейти к анализу с этими предикторами»** — они автоматически подставятся как обязательные на странице «Анализ и отчёт» (в полу‑автоматическом режиме).

## Расчёт прогнозируемых значений

Раздел **«Расчёт прогнозируемых значений»** применяет ранее обученную модель, чтобы оценить целевой показатель по сопутствующим измерениям. Есть два равноправных способа задать данные — результат и выгрузка в обоих одинаковые.

> Расчётные значения — это оценки модели, а не лабораторные измерения. Они помечаются `ValueSource=estimated`, `IsEstimated=true` и не перезаписывают исходные данные.

**Шаг 1 (общий). Выберите пакет модели** из списка обученных (дата, целевой показатель, сценарий, признак «ML лучше базовой»). Если пакетов нет, сначала постройте отчёт.

**Шаг 2. Выберите способ ввода данных** переключателем:

### Способ А — загрузить файл

Когда есть выгрузка, в которой целевой показатель отсутствует, но измерены сопутствующие показатели.

1. **Загрузите файл** с пропусками целевого показателя.
2. **При необходимости настройте параметры:** минимум наблюдаемых признаков, минимальное покрытие признаков, разрешение на отсутствующие столбцы признаков (заполнятся пустыми значениями), оценку всех строк.
3. **Запустите оценку.**

### Способ Б — ввести значения вручную

Когда файла нет, а нужно быстро оценить показатель по нескольким значениям «на руках».

1. Программа покажет, какие показатели ожидает модель, и таблицу для ввода: столбец **«Дата отбора»** и по столбцу на каждый показатель. Заполните значения; можно добавить несколько строк — это несколько отдельных проб.
2. **Дата отбора обязательна.** Для сезонных моделей она влияет на оценку (сезон выводится из даты так же, как при загрузке файла), поэтому указывайте реальную дату пробы.
3. При необходимости настройте пороги (по умолчанию достаточно одного заполненного показателя).
4. Нажмите **«Рассчитать»**.

В обоих способах результат одинаков: число строк для оценки, успешно оценённых и пропущенных; таблица предсказаний с выгрузкой в **CSV/XLSX** (как если бы на входе был файл) и понятная диагностика — почему строки были пропущены и что с этим делать.

## Просмотр и удаление отчёта

Раздел **«Просмотр отчёта»** открывает ранее построенные отчёты из папки `reports/` — они перечислены в формате «дата — сценарий — целевой показатель». Содержимое показывается теми же вкладками, что и при первом запуске.

Внизу страницы отчёт можно безвозвратно удалить: сначала подтвердите действие галочкой, затем нажмите «Удалить отчёт».

## Как интерпретировать результаты

**Пригодность данных:**

- `suitable` — данных достаточно для осторожного моделирования;
- `weakly_suitable` — модель можно обучить, но ограничения существенны, оценки трактуйте осторожно;
- `unsuitable` — обучение заблокировано. Это не ошибка: данных для надёжной модели мало. Используйте паспорт и корреляции как самостоятельный диагностический результат.

Частые причины ограничений: целевой показатель отсутствует, мало наблюдений, много пропусков, много цензурированных значений, мало совместных измерений цели и признаков, мало сопутствующих показателей.

**ML‑модель не лучше базовой.** Это честный результат, а не сбой. Базовая модель — простой ориентир (медиана, последнее значение). Если ML его не превосходит, отчёт сообщает об этом прямо, и расчётные оценки следует трактовать с повышенной осторожностью.

**Как выбирается лучшая модель.** Сравнение идёт не по одному финальному тесту, а по комбинированной оценке: 40 % — holdout (самый свежий период), 60 % — backtest (несколько прошлых окон). Это снижает риск выбрать модель, случайно угадавшую последний период, но неустойчивую на остальных.

**Стабильность** — отношение метрики backtest к holdout:

- около 1 — модель ведёт себя одинаково на всех проверках;
- заметно больше 1 — признак переобучения или дрейфа данных;
- заметно меньше 1 — финальный период был нетипичным; реальная точность ближе к backtest.

Если данных для скользящей проверки мало, программа переходит в запасной режим и сообщает об этом.

## Типичные проблемы

**«Выбранные фильтры не соответствуют ни одному срезу данных».** Указанный ОКТМО или тип точки отсутствует в файле. Сверьтесь с превью и выберите значения из реального списка.

**«Missing required raw columns».** В файле нет обязательных столбцов. Проверьте формат выгрузки и при необходимости выберите профиль источника вручную.

**Статус `unsuitable`.** Данных недостаточно для модели в этом срезе. Причины — в блоке «Причины и ограничения». Расширьте период, ОКТМО или тип точки, если это предметно допустимо.

**Кракозябры при открытии CSV.** Откройте CSV в Excel с кодировкой UTF‑8 (Данные → Из текстового/CSV → UTF‑8) либо используйте лежащий рядом файл XLSX с готовыми русскими заголовками.

**Не понятны столбцы в таблицах в результатах анализа.** У каждой таблицы есть XLSX-версия, более приятная глазу, лежащая в той же папке.

## Командная строка

Графический интерфейс закрывает все основные задачи, но программа полностью работает и из командной строки — это удобно для автоматизации и повторяемых запусков:

```powershell
water-analysis report `
  --input data/raw/main.xlsx `
  --scope drinking_water_combined `
  --oktmo 30000000001 `
  --target "Жесткость общая" `
  --output-dir reports\my_report
```

Полная справка: `water-analysis --help` и `water-analysis <команда> --help`. Перечень команд и структура результатов описаны в `README.md`.
